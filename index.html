Polygonize
This script runs through all scan data in the current project then:
  1. Polyginizes
  2. Fills holes
  3. Thins
  4. Removes data that is not connected to the model based on number of connected triangles being less than 10% of all triangles
  5. Splits the model on the Y-axis
  6. Exports all models individually in an folder in the current project
All of these features are fully customizable and the default values are based on the size of the model scanned. This script has saved the department several hours per project in post-processing and consistency of model quality throughout the department.

Duplicate Scans
Out of all of my scripts for this software this is single-handedly the most useful and most time-saving script I have written. It is extremely simple and extremely useful. Features include
  1. Creates a copy of the selected scan
  2. Pastes that copy at the end of the current measurement series
  3. Changes the scanning parameters that was used to grab more points to fill in gaps in the scan
This script is awesome for stubborn sections of a model that the scanner just refuses to collect data. I realized one day that when the model is scanned the scan data itself is all collected it's only the filters that show you Atos built-in quality points instead of all points collected. This script gives you both the best points and the extra points and when you use the polygonize script later it will choose the best points available and fill in the rest where needed. This eliminates most fighting with the scanner and picks up all points without having to constantly re-scan sections of the model.


Intersect Curve
This script is great for inspection on known targets. It requires a little bit of set-up which I will explain at the end of this section. Features include:
  1. Finds the theoretical intersection of two curves
  2. Creates a distance line of length 0 on the intersection
  3. Creates an actual point on that line
  4. Creates a nominal point based on the name of the two curves
  5. Compares the nominal point to the actual point
  6. Sets the display of the label to show the comparison with an indicator showing how close they are to being within a 5mm tolerance
To run this script you need to create 2 curves on the model that intersect at a known target name them both with the coordinates for the target location. For example: X1000_Y1000_Z1000_1, X1000_Y1000_Z1000_2. When you run this script it will create the nominal point at X 1000 Y 1000 Z 1000, and compare it to the actual intersection point between the two curves. In Atos, there is no direct way to find the intersection of 2 curves you must use lines which is highly inaccurate considering most models are not flat making lines at the intersection not be on the model itself but several millimeters off the model. That is why this script is needed because curves are always on the model making the intersection also being on the model.


Single-Player Mode
This is a fun script that automates most of the scanning for you as long as it is set up to a stable tower. Features include:
  1. Takes a scan pass every 10 seconds
  2. If there are any errors (movement, lighting, unknown targets), it waits an additional 15 seconds and tries again
This script makes for a great workout, 10 seconds is not very much time to move a tower to the next scan location. I do have Easy at 20 seconds, Medium at 15 seconds, and Hard mode with 10 seconds between shots. I've often toyed around with the idea of making a game out of this script where you start off with 10,000 points and every shot takes 100 points off and every error 1000 points off but never implemented it. This script is great if you are working alone and just need to scan fast and do not need to check the system between scan passes.

Auto-setup
This script is hands down the most complicated script I have written for Atos. It automatically creates a 3-2-1 transformation on the scan project based on your selection of reference points. Features include:
  1. Creates a table based on selected reference points
  2. Sorts through those points using Linear Algebra to find a 4-point plane for the Z plane
  3. Finds 12mm points to use for the Y-axis
  4. Chooses one of the 12mm points to use for the X-point
  5. Finds the distance between the Z plane and the model and sets it up for the transformation
  6. Sets the Y-axis to 508mm
  7. Sets the X-point to 730mm
This script is still in experimental mode and currently works 90% of the time. The only issue is which direction the Y-axis is pointing which can only be set visually by looking at the model and deciding if it is oriented correctly. This error is easily fixable by a human and without machine learning and visualization software available in the program it is most likely impossible to guarantee correct Y-axis direction. All numbers are based on a surface plate with plugs every 127mm. It uses Python classes to build the points in a format used to create the points, lines, plane, and make them ready to use for the 3-2-1 transformation.
